# Trace Export: Performance and Profiling

The `glassbox profile` command exports execution traces as performance profiles
and gas-usage reports. This document describes its export modes, validation
checks, error handling, and troubleshooting guidance.

---

## Overview

`glassbox profile` has two modes:

| Mode | Flag | Output |
|------|------|--------|
| **File mode** | `--file <trace-file>` or positional arg | pprof profile (`.pb.gz`) |
| **Live mode** | `--xdr <envelope>` | Gas optimization report (+ optional JSON) |

Both modes share the same pre-flight validation layer — all path and input
problems are surfaced before any expensive work begins.

---

## Pre-flight Validation

All flag validation runs in `PreRunE` before the trace is read or any
simulation is started.

### Input source (required)

Exactly one input source must be provided:

```
glassbox profile execution.json          # positional trace file
glassbox profile --file execution.json   # explicit flag
glassbox profile --xdr AAAAA...==        # live simulation from XDR
```

**Error when no input is provided:**
```
no input provided — specify one of:
  --xdr <file-or-base64>   simulate from a transaction envelope
  --file <trace-file>      analyze an existing trace file
  <trace-file>             positional trace file argument

Run 'glassbox profile --help' for full usage
```

**Error when both `--xdr` and `--file` are given:**
```
--xdr and --file (or a positional trace-file argument) are mutually exclusive;
provide only one input source
```

### `--network` (live mode)

**Valid values:** `testnet`, `mainnet`, `futurenet`

**Error for invalid network:**
```
invalid --network "devnet"; must be one of: testnet, mainnet, futurenet
```

### Trace file existence (file mode)

The trace file is verified to exist and be accessible before parsing begins.

**Error when file is missing:**
```
trace file not found: "execution.json"
Tip: generate a trace first with 'glassbox debug --save-snapshots <file>'
```

### `--output` (pprof output path)

The pprof output path is validated when it differs from the default (`profile.pb.gz`):

- Must not end with `/` or `\` (no bare directory paths)
- Parent directory must exist

**Error for directory path:**
```
--output "./profiles/" looks like a directory path; provide a full file path
  Fix: specify a filename (e.g. --output ./profiles/gas.pb.gz)
```

**Error for missing parent directory:**
```
--output directory "./profiles" does not exist; create it before running
  Fix: run 'mkdir -p ./profiles' or choose a path in an existing directory
```

### `--out-json` (JSON report path)

The `--out-json` path receives the same validation as `--output`:

- Must not end with `/` or `\` (no bare directory paths)
- Parent directory must exist

**Error for directory path:**
```
--out-json "./reports/" looks like a directory path; provide a full file path
  Fix: specify a filename (e.g. --out-json ./reports/gas-report.json)
```

**Error for missing parent directory:**
```
--out-json directory "./reports" does not exist; create it before running
  Fix: run 'mkdir -p ./reports' or choose a path in an existing directory
```

## Execution-Phase Validation and Diagnostics

### Empty-trace handling

When a trace file contains no execution steps, `glassbox profile` still writes
a valid (empty) pprof profile rather than failing. A warning is emitted so
users understand why the flamegraph is blank:

```
Warning: trace contains no execution steps — the pprof profile will be empty.
  Possible causes:
    - The simulation ran successfully but produced no diagnostic events
    - The trace file was generated by an older simulator version
  Tip: re-run with 'glassbox debug --save-snapshots <file>' to capture a
       fresh trace, or use 'glassbox profile --xdr <tx.xdr>' for a live
       report with full gas budget data
```

**Note:** The profile file is still written and the command exits successfully.
A note is printed after the `[OK]` line to remind you the profile is empty.

### Step-index corruption

If the trace has mismatched step indices (indicating data corruption), the
command fails with an actionable message:

```
trace step index mismatch at position 1: state.Step=5 — trace may be corrupted
  Fix: re-run 'glassbox debug --trace-output <file>' to regenerate a clean trace
  Check: if using --save-snapshots, ensure no partial writes occurred
```

### File-write failure

If the output file cannot be created (e.g. permission denied):

```
failed to create output file "gas.pb.gz": open gas.pb.gz: permission denied
  Fix: check that the directory exists and you have write permission
  Tip: use -o to specify a different output path (e.g. -o /tmp/gas.pb.gz)
```

### pprof generation failure

If the pprof profile cannot be generated from the trace data:

```
failed to generate pprof profile: <reason>
  Fix: re-run 'glassbox debug --save-snapshots <file>' to regenerate a clean trace
  Check: run 'glassbox doctor' to verify simulator compatibility
```

## Zero-gas Traces

Steps that report `gas_used = 0` (or no `gas_used` key in `HostState`) are
filtered from pprof samples. This is intentional — zero-gas steps don't
contribute to the flamegraph and are omitted to keep profiles clean.

If **all** steps report zero gas, the profile will be empty. The pprof file is
still written but a hint is printed on stderr:

```
Info: no gas samples found in trace — all steps reported zero gas usage.
  The pprof profile will be empty. Possible causes:
    - The contract does not record gas in HostState["gas_used"]
    - The simulator version predates gas tracking
  Tip: use 'glassbox profile --xdr <tx.xdr>' for a live gas report
```

## Live Simulation Mode (`--xdr`)

Live mode runs a fresh simulation and returns an optimization report without
requiring a pre-existing trace file.

```sh
glassbox profile --xdr <tx.xdr> --network testnet
glassbox profile --xdr <base64-envelope> --network mainnet --out-json report.json
```

The live report includes:
- **Overall efficiency score** (`OverallEfficiency`)
- **CPU / memory budget breakdown**
- **Per-category cost table**
- **Optimization tips** with severity levels (High / Medium / Low)

Exporting as JSON with `--out-json` enables CI integration and gas-regression
tracking across builds.

### Simulator compatibility error

If the simulator does not return an optimization report:

```
simulator did not return an optimization report
This may indicate an unsupported protocol version or a simulator version mismatch;
run 'glassbox doctor' to check compatibility
```

## Flamegraph Output (`--profile` flag in `glassbox debug`)

The `glassbox debug` command can generate an interactive flamegraph HTML page
in addition to the standard pprof profile. Enable it with `--profile`:

```sh
glassbox debug --profile --profile-format html <tx-hash>
```

**Valid `--profile-format` values:** `html` (default), `svg`

**Error for unsupported format:**
```
invalid --profile-format "pdf" — must be one of: html, svg
  Fix: use --profile-format html (interactive flamegraph) or svg (raw SVG)
```

When the flamegraph is generated for an empty trace, the HTML page is still
written but shows no frames. A warning is printed to stderr:

```
Warning: trace contains no execution steps — flamegraph will be empty.
  Possible causes:
    - The simulation produced no diagnostic events
    - The trace was captured by an older simulator version
  Tip: use 'glassbox profile --xdr <tx.xdr>' for a live gas report,
       or re-run with 'glassbox debug --save-snapshots <file>'
```

## Analyzing pprof Profiles

After generating a profile, use the standard Go pprof toolchain:

```sh
# Interactive web UI (recommended)
go tool pprof -http :9090 gas.pb.gz

# Top functions by gas usage
go tool pprof -top gas.pb.gz

# Flame graph in terminal
go tool pprof -flame gas.pb.gz
```

## Best Practices

### 1. Always specify an output directory explicitly

```sh
mkdir -p ./profiles
glassbox profile execution.json -o ./profiles/$(date +%Y%m%d)-gas.pb.gz
```

### 2. Use live mode for the most accurate gas data

```sh
glassbox profile --xdr ./envelope.xdr --network testnet --out-json ./report.json
```

### 3. Track gas regressions in CI

```sh
# Export as JSON and compare to a baseline
glassbox profile --xdr $XDR --network testnet --out-json ./ci-report.json
jq '.OverallEfficiency' ./ci-report.json
```

### 4. Combine with `glassbox trace` for full analysis

```sh
# First get the detailed trace
glassbox debug --save-snapshots ./execution.json $TX_HASH

# Then profile gas usage
glassbox profile ./execution.json -o ./gas.pb.gz

# And open the interactive trace viewer
glassbox trace ./execution.json
```

## Troubleshooting

### "trace contains no execution steps"

**Cause:** The trace file exists but has no steps recorded.

**Solutions:**
1. Re-run `glassbox debug --save-snapshots <file>` to generate a fresh trace
2. Use `glassbox profile --xdr <tx.xdr>` for a live report
3. Verify the transaction actually executed on-chain

### "trace step index mismatch"

**Cause:** The trace data is corrupted — step indices don't match array positions.

**Solutions:**
1. Delete the trace file and re-run `glassbox debug --trace-output <file>`
2. Check for filesystem issues if using `--save-snapshots`

### "simulator did not return an optimization report"

**Cause:** Simulator version mismatch or unsupported protocol version.

**Solutions:**
1. Run `glassbox doctor` to check simulator compatibility
2. Update the simulator binary
3. Verify the XDR envelope targets a supported protocol version

### "no gas samples found in trace"

**Cause:** All steps report `gas_used = 0` or the key is absent from `HostState`.

**Solutions:**
1. Use `glassbox profile --xdr <tx.xdr>` for live gas data
2. Verify the contract records gas in `HostState["gas_used"]`
3. Check that the simulator version supports gas tracking
